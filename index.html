WPilot A space journey in your browser

Welcome to WPilot (Beta). WPilot is a browser-based, multiplayer, space shooter, that is inspiered by old classic such as XPilot, Thrust and Gravity Force.

The game is a "proof-of-concept" project, where are trying too see whats possible in browser without using any kind of third-party plugins. The game is completely built using new HTML5 technologies. Read more

Gameplay

The gameplay is pretty simple. The basic is: avoid to be shoot, while shooting your opponents. The first player that reaches the score limit wins the round.

Controlling your Vessel

Your ship is easy to maneuver, but is hard to master. The up-arrow will bring your ship forward. Use your left- and right-arrow to rotate your ship left and right. Your only defence against walls, and opponent bullets is your shield, use down-arrow to activate. While activated, you are indestructible. The shield isn't free of charge though, your energy will constantly decrease while activated.

Keyboard bindings

There are two ways to demolish your opponent. You can either nock your opponent out with your shield, or you can shoot him/her with your canon by pressing the space-key. Each bullet cost you energy as well, so you have to balance the use of both Shield and Canon.

The Warmup round

Each game starts with a warm-up round. The warm-up let's you to get a feel about your opponents.

When you are ready for some real action, press the r-key. When enough players is ready, the game starts.

Powerups

What would a space-shooter be without compelling enhancements? WPilot contains three kinds of destructive powerups, each improving your veesel's boomstick. It's extra powerful when combinding them with each other

  • Rapid fire increases your fireing speed and lower the energy cost of each bullet.
  • Ricochet gives each bullet "wallet-bounce" abillities, they become more deadly when bouncing several times against the walls.
  • Spread fire fires three bullets instead of one. This powerup is extremly deadly when combinded with one of the other two powerups.

Winning a round

Each player scores when blowing an opponent up. You will loose score when taking suicide. The first player that reaches the round limit wins the game.

Talking to other players

You can taunt, chat or just salute your opponents by pressing the enter-key. The prompt can also be used to issue commands. For example, you can change your name in-game by typing "/name my_new_name" (without the quotes).

Maps

WPilot contains a total of four official maps. You can also great your own map with the built-in map-format.

Battle Royale (6-8 players)

This is the built-in, standard map. Search and destroy your opponent in the outer passage or take a chance for the powerups in middle.

Browser compatibility

WPilot is unfortunately not playable in all browsers. This compatibility list helps you to find a browser that is supported by WPilot.

The browser rank gives you an indication of how well the game perform.

The Recommended indicates that the browser is fully supported. It can be some quirks but nothing major.

The Not recommended indicates that the game runs,but that it's one or more major quirks that causes the game to run badly. Read the description for more information.

The Not supported indicates that the game won't run at all. There may be support for the browser in a future release, but guarentees are omitted.

  • Internet Explorer - Win Not supported

    Internet Explorer is NOT support by WPilot. WPilot depends up on Canvas which Internet Explorer lacks. There is third-party solution that emulates the canvas object, but they are really slow. IE9 may be supported in feature, if the team behind (which rumors are pointing at) the browser decides to implement canvas support.

  • Safari 4 - Mac Recommended

    Safari delivers a good FPS rate. Safari does however lack support for WebSocket, so it's depends upon the Flash fallback plugin. The experience is for the most good, but sometimes, the server latency goes up really high.

  • Webkit (nightlies) - MacRecommended

    Webkit delivers outstanding network performance, low latency and no lost packets. The FPS is not as good as for Safari (~30% lower FPS) but the game should play well. A tip is to disable in-game sound effects, which increases frame rate slightly.

  • Firefox 3.6 - MacNot recommended

    Firefox for Mac delivers a really good FPS rate. It even outstand Safari in our tests. Firefox also lack support for WebScoket, so Flash is a requirement. It seems to be some kind of problem with the Flash implemention though. The ping is very high, even when connecting to localhost.

  • Google Chrome - Win/MacNot recommended

    Google Chrome has some issues with WPilot, both on Windows and on Mac: FPS performance is poor, sounds stops playing after a while, some issues with text rendering in some situations. However, the game is playable, but you will notice the lag. It's highly recommended to turn of sounds before playing, the game will else crash after a couple of minutes.

  • Firefox 3.6 - WinRecommended

    Firefox for Mac delivers a really good FPS rate. It even outstand Safari in our tests. Firefox also lack support for WebScoket, so Flash is a requirement. It seems to be some kind of problem with the Flash implemention though. The ping is very high, even when connecting to localhost.

  • Opera 10.50 - Mac/WinNot supported

    Only tested on Mac. Opera have some trouble communicating with the Flash fallback plugin. The game connects to the server 1 of 10 times. No error messages is displayed so the problem is difficult to debug. There also seems to be some kind of problem with the underlying text rendering interface. Some texts are not rendered at all. How ever, the FPS seems to be really good, but the game is not playable.

Project goal

Yesterday's browser wasn't built to handle FPS intense applications or low-latency game network protocols.

This is about to change, dramatically. The browser standard factory is currently cooking up some very interesting new features for our browsers. New HTML5 technologies such as Canvas, WebSocket and WebGL, will change the way we thing of applications in the browser.

The goal of this projects was/is to test how far we can get using the browser, this with out using any third-party plugins. This game is made with pure browser technologies, no Flash (Not entirely true though, Flash is used to emulate the WebSocket interface in browsers that lacks the feature), no Silverlight and no Unity3D.

As you can see, the results so far, is pretty impressing. The game engine runs at 60fps sec. Each game tick handles collision data, movement and game rules. The server constantly sending position updates to each client which requires a low-latency connection between the two parties.

The native implementation of WebSocket (In Safari and Google Chrome) handle this very well. The fallback plugin is kind of unstable on the mac but seems to do a good job on Windows machines.

WPilot, and similar applications, will be more stable and faster, when the browser vendors starts implementing new, exciting, features listed in the HTML5 draft.

Running your own server

So, you are interesting in setting up a server of your own. Thats great! The entire server is built in something called node.js. Node.js is a framework built on top of Googles lighting fast Javascript engine V8

First, download and install node.js (see instructions on project homepage). Then, download the latest version of WPilot. Extract the zip to directory of choice. All server configuration is done via a set of command-line switches. You can get list of all available switches by typing:

$ ./wpilots.js -H

You probably want to change the server name and port. Use the --name switch to change name and the --port to change the TCP port to listen on.

Map cycling

WPilot comes with a set of maps. All maps is located in the the "$INSTALLDIR/maps" directory. You can choose to circle between multiple maps at starup. Use the --map for each map you want to play. It's also possible to change map in-game.

In-game administration

As an administrator, you can change maps, kick players in-game. You first have to start the server with the switch "--admin_password secret password" (where secret password is your password), this enabled the in-game admin mode. To authorize you in-game, type:

/sv_password secret password

After authorization, you can issue admin commands. For example, the command:

/sv_map maps/battle_royale.json

will change map to Battle Royale. You can also kick unwanted players with:

/sv_kick  player_name reason

Note: The reason argument MUST be present in order to kick someone.
