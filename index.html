WPilot A space journey in your browser

Welcome to WPilot (Beta). WPilot is a browser-based, multiplayer, space shooter, that is inspiered by old classic such as XPilot, Thrust and Gravity Force.

The game is a "proof-of-concept" project, where are trying too see whats possible in browser without using any kind of third-party plugins. The game is completely built using new HTML5 technologies. Read more

Gameplay

The gameplay is pretty simple. Avoid to be shoot, while shooting your opponents.

Your shield is your only defence, use it wisely. Your energy meter goes down when using shield and while firing your canon. The energy meter recovers pretty quickly but sometime that isn't enough.

Powerups

The game contains three kinds of powerups. Each powerup improves your ships canon. The powerups can be combind for even more powerful boomstick.

Rapid fire increases your fireing speed and lower the energy cost of each bullet.

Ricoschet bullets gives each fired bullet ricoschet abillities. They become more deadly when bouncing on the walls.

Spread fire fires three bullets instead of one. This powerup is extremly deadly when combinded with one of the other two powerups.

The goal of this projects was to test how far we could push the browser, this with out using any third-party plugins. This game is made with pure browser technologies, no Flash (Not entirely true though. We are using Flash as fallback, in browser that lacks support for WebSocket.), no Silverlight and no Unity3D.

  • Internet Explorer - Win

    Internet Explorer is NOT support by WPilot. WPilot depends up on Canvas which Internet Explorer lacks. There is third-party solution that enables a canvas like object, but they are really slow. IE9 may be supported in feature, if the team behind the browser decides to implement canvas support.

  • Safari 4 - Mac

    Safari delivers a good FPS rate. Safari does however lack support for WebSocket, so it's depends upon the Flash fallback plugin. The experience is for the most good, but sometimes, the server latency goes up really high.

  • Webkit (nightlies) - Mac

    Webkit delivers outstanding network performance, low latency and no lost packets. The FPS is not as good as for Safari (~30% lower FPS) but the game should play well. A tip is to disable in-game sound effects, which increases frame rate slightly.

  • Firefox 3.6 - Mac

    Firefox for Mac delivers a really good FPS rate. It even outstand Safari in our tests. Firefox also lack support for WebScoket, so Flash is a requirement. It seems to be some kind of problem with the Flash implemention though. The ping is very high, even when connecting to localhost.

  • Google Chrome - Win/Mac

    Google Chrome has some issues with WPilot, both on Windows and on Mac: FPS performance is poor, sounds stops playing after a while, some issues with text rendering in some situations. However, the game is playable, but you will notice the lag. It's highly recommended to turn of sounds before playing, the game will else crash after a couple of minutes.

  • Firefox 3.6 - Win

    Firefox for Mac delivers a really good FPS rate. It even outstand Safari in our tests. Firefox also lack support for WebScoket, so Flash is a requirement. It seems to be some kind of problem with the Flash implemention though. The ping is very high, even when connecting to localhost.

  • Opera 10.50 - Mac

    Only tested on Mac. Opera have some trouble communicating with the Flash fallback plugin. The game connects to the server 1 of 10 times. No error messages is displayed so the problem is difficult to debug. There also seems to be some kind of problem with the underlying text rendering interface. Some texts are not rendered at all. How ever, the FPS seems to be really good, but the game is not playable.

Background

Yesterday's browser wasn't built to handle FPS intense games or low-latency network packets.

This is about to change, dramatically. The browser standard factory is currently cooking up some very interesting new features for our browsers. New HTML5 technologies such as Canvas, WebSocket and WebGL, will definetly change the way we thing of applications in the browser.

The goal of this projects was to test how far we could push the browser, this with out using any third-party plugins. This game is made with pure browser technologies, no Flash (Not entirely true though. We are using Flash as fallback, in browser that lacks support for WebSocket.), no Silverlight and no Unity3D.

Server setup

So, you are interesting in setting up a server of your own. Thats great! The entire server is built in something called node.js. Node.js is a framework built on top of Googles lighting fast Javascript engine V8

First, download and install node.js (see instructions on project homepage). Then, download the latest version of WPilot. Extract the zip to directory of choice. All server configuration is done via a set of command-line switches. You can get list of all available switches by typing:

$ ./wpilots.js -H

You probably want to change the server name and port. Use the --name switch to change name and the --port to change the TCP port to listen on.

WPilot comes with a set of maps. All maps is located in the the "$INSTALLDIR/maps" directory. You can choose to circle between multiple maps at starup. Use the --map for each map you want to play. It's also possible to change map in-game.
